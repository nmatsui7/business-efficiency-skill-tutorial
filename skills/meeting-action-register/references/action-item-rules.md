# Action Item Extraction Rules

## Owner identification

- Use the exact name as it appears in the notes.
- If a name appears with a title (e.g., "Sarah, Project Manager"), extract only the name.
- If the notes say "the team" or "someone", mark the owner as "Unassigned."
- If multiple people are mentioned for one task, use the person most directly responsible.

## Deadline interpretation

- Exact dates: Use the date as stated (e.g., "by Friday" becomes "Friday").
- Relative dates: Use the phrase as stated (e.g., "by end of day today").
- Vague phrases: Use the phrase without interpreting (e.g., "soon" stays "soon").
- No deadline: Write "No deadline stated."

## Priority rules

- **High**: The task blocks other work, has a near-term deadline (within the week), or was explicitly called out as urgent.
- **Medium**: Standard follow-up tasks with moderate deadlines or general importance.
- **Low**: Tasks with flexible timing, nice-to-have items, or items with no deadline.

## Decision identification

- Look for explicit decision markers: "Decision:", "We decided", "The group agreed", "It was resolved".
- Capture the full decision in one sentence.
- If the notes describe an outcome without an explicit marker, treat it as a decision if it implies a confirmed course of action.

## Open question identification

- Look for questions without assigned owners.
- Look for topics deferred to future discussion.
- Look for "TBD", "to be determined", or "open question" markers.
